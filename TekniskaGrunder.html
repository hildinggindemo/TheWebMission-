Hem Relevanta Lagar Tekniska Grunder

Teknsika Grunder

OSI-modellen:

OSI-modellen används för att förklara hur kommunikation fungerar i nätverk. Den består av sju lager som tillsammans hanterar allt från den fysiska överföringen av data till hur information visas för användaren.

Physical (fysiskt lager): Ansvarar för den faktiska överföringen av data, till exempel genom kablar eller fiber.

Data Link: Delar upp data i “frames” och skickar dem inom ett lokalt nätverk. Här sker även felkontroll och hantering av MAC-adresser.

Network: Ansvarar för adressering och styr datapaket mellan nätverk med hjälp av IP-adresser.

Transport: Ser till att kommunikationen är pålitlig, delar upp data i segment och säkerställer att informationen når fram, till exempel via TCP.

Session: Skapar separata kommunikationssessioner för olika program med hjälp av portnummer, som port 80 för HTTP.

Presentation: Bestämmer hur data ska formateras och struktureras så att både avsändare och mottagare förstår varandra.

Application: Lagret där användaren interagerar, till exempel via en webbläsare som skickar och tar emot HTTP-förfrågningar.

Tillsammans visar OSI-modellen hur data rör sig från fysisk överföring till applikation.


Server–klient-modellen:

Server–klient-modellen visar hur kommunikation sker på internet.

Server: En dator eller tjänst som tillhandahåller resurser, information eller funktioner.

Klient: En enhet som begär dessa resurser, ofta en webbläsare. När en användare besöker en webbsida skickar klienten en förfrågan till servern, som sedan svarar med exempelvis en webbsida, en bild eller data från en databas.

Modellen används i många sammanhang, till exempel HTTP/HTTPS för webbsidor, SMTP för e-post och FTP för filöverföring.

Internet Protocol och IP-adresser:

IP (Internet Protocol) bestämmer hur data skickas, tas emot och identifieras över ett nätverk. Varje enhet får en unik IP-adress. Det finns två typer: IPv4 (32 bitar, t.ex. 192.168.1.1) och IPv6 (128 bitar, hexadecimal notation). IP-adresser kan vara publika eller privata. Routrar skickar paketen längs den mest effektiva vägen, och vid behov delas stora paket upp i mindre delar.

DNS – Domain Name System:

DNS fungerar som internets telefonbok. Den översätter domännamn, som google.se, till rätt IP-adress. Webbläsaren använder denna information för att kontakta rätt server. DNS gör internet enklare att använda eftersom man slipper memorera långa nummer.

HTTP och HTTPS:

HTTP används för att överföra information mellan webbläsare och webbservrar. Klienten skickar en request och servern svarar med en response som innehåller HTML, bilder eller annan data. HTTP är inte krypterat, så trafiken kan avlyssnas. HTTPS är den säkra versionen som använder SSL/TLS för att kryptera trafiken och skydda information som inloggningar eller betalningar.
